# Compliance Document

## Objectives
To ensure that all configurations and practices adhere to compliance requirements.

### Rule 9. Remote state
Utilize a remote S3 backend for storing Terraform state to enhance security and accessibility.

### Rule 10. Use module
Utilize the branch-environment module to streamline deployments effectively.

### Rule 11. Min Aurora capacity
Ensure that Aurora Serverless v2 is configured with the minimum viable capacity settings for optimal performance.

### Rule 12. Circuit breakers
Implement ECS deployment circuit breakers to manage failures gracefully and maintain application availability.

### Additional Rules
Please ensure all rules are followed as per the compliance guidelines.
